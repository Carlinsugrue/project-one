Code Quality

3/5

Due to my projects nature, I haven’t particularly written much code, although I have produced several scripts for the lingo-builder server and the website itself. My strongest technical capabilities come from my Unix knowledge and OS understanding, which helped greatly in the BIT Platform group. Because of my chosen study path leaning more towards the world of systems administration my coding abilities have left my PHP/JavaScript pages suboptimal, but I believe I’m improving!


Code extract from lingo-builder / calling server-side scripts

Following Development Best Practices

3/5

I feel that when it comes to best practices, I try my best to make things I develop simple and user friendly whilst also testing in a few various environments. A good example may be me always implementing new updates for CodeLingo software such as the platform or lingo binaries on my own home machine by just utilizing VMWare Workstation. I can then diagnose any potential problems in a well understood environment before pushing to the servers located at OP. However, I understand that one of my largest downfalls is not regularly using industry recognized version controls such as git.

Version Control

3/5

Not the best. I recognize that I implemented proper version control a bit late. As when I was developing my projects it was only me utilizing the code or software, so I only employed the 2 separate backups in 2 different locations approach. Usually I would have my software stored on my personal machine, external hard drive and project machine but as the end drew near I started to employ the use of git.
